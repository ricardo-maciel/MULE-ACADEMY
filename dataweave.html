O Cérebro da Integração

“Lembra que sistemas falam idiomas diferentes (JSON, XML, CSV), e que precisam de um tradutor para se comunicarem?
Agora você vai aprender a linguagem usada pelo tradutor.”

O que é o DataWeave?

Imagine que o integrador de sistemas é uma pessoa poliglota. Para fazer seu trabalho, ela precisa de uma ferramenta mental de tradução rápida.

DataWeave é a linguagem desenvolvida especificamente para isso. Ela pega uma informação em um formato (como JSON), processa as regras de negócio que você definir e entrega os dados prontinhos em outro formato (como XML ou CSV).

Transformação Dinâmica

Ele não apenas altera a extensão do arquivo; ele reorganiza as chaves, calcula valores e filtra o que não é útil.

📦 O que é o Payload?

Imagine um caminhão de entregas viajando pela rodovia. O caminhão em si é a infraestrutura de rede — os servidores, a internet, os cabos. Mas o que realmente importa para quem vai receber a mercadoria é a caixa de encomendas guardada no baú. Essa caixa é a carga útil.

Na tecnologia, chamamos essa carga de dados de Payload. É o conteúdo que trafega de um sistema para outro — a informação real que precisa ser entregue e processada.

A Palavra Mágica do DataWeave

Sempre que você escreve payload em um script, você está dizendo: "Ei, pegue a caixa de encomendas que acabou de chegar e leia o que está dentro!"

AULA 2 — HELLO WORLD

Sua Primeira Tradução

Todo script DataWeave tem um cabeçalho (que define a versão e o formato de saída) e o corpo da transformação.

Estrutura Básica
%dw 2.0                     // 1. Cabeçalho: Versão do tradutor
output application/json     // 2. Formato: Qual idioma será a saída
---                         // 3. Divisor: Separa o cabeçalho das regras
payload.message             // 4. Corpo: O que traduzir (payload é o dado de entrada)

Laboratório Prático: Hello World

Entrada (JSON)
{
  "nome": "Maria"
}
Script DataWeave
%dw 2.0 output application/json ---
{ name: }
Saída Esperada (JSON)
{
  "name": "Maria"
}

AULA 3 — MUDANDO IDIOMAS

Traduzindo Formatos

Mudar o tipo de saída no DataWeave é tão simples quanto alterar o cabeçalho `output application/...`. Veja a mágica acontecer!

application/json

Moderno, baseado em chaves e valores. Idioma padrão web.

application/xml

Estrutura hierárquica baseada em tags (<tag>).

application/csv

Linhas separadas por vírgula, ideal para tabelas.

Laboratório: Conversão de Formatos

Entrada (JSON)
{
  "nome": "João"
}
Script DataWeave
%dw 2.0
output

AULA 4 — REGRAS DO TRADUTOR (LÚDICO)

Ferramentas do Tradutor

Para transformações complexas, o tradutor possui operadores que funcionam como ordens humanas inteligentes:

map = “Lista Completa”

Pega uma lista inteira de dados (como clientes) e traduz cada item um por um para uma nova lista de saída.

filter = “Escolher o Importante”

Varre a lista e ignora tudo o que não encaixar na sua condição. Retorna apenas o que for importante.

if/else = “Tomar Decisões”

Faz perguntas aos dados: se a condição for verdadeira, traduz de um jeito; se for falsa, traduz de outro.
