MÓDULO 2 — CONCEITOS INICIAIS

Como os sistemas organizam informações?

Antes de entender linguagens complexas de dados como JSON ou XML, você precisa entender o conceito mais simples de todos: **o que é um Objeto**.

CAPÍTULO 1

A Menor Caixinha de Dados: O Objeto

No mundo real, tudo é um "objeto" que possui características específicas. Uma **Pessoa** possui nome, idade e endereço. Um **Carro** possui marca, modelo e cor.

Na programação, fazemos a mesma coisa! Criamos **Objetos Digitais** que são "caixinhas virtuais" contendo essas mesmas características (que chamamos de **atributos** ou **propriedades**).

"Um Objeto Digital é a forma que os sistemas usam para carregar informações sobre itens do mundo real de uma forma que o computador consiga compreender."

Pessoa Real vs Objeto Digital

João da Silva

Humano Físico

Mora em **São Luís** e tem **25 anos** de idade.

Objeto: Pessoa
Nome: "João"
Idade: 25
Cidade: "São Luís"
CAPÍTULO 2

Representando o Objeto em Texto

Como as caixinhas de dados são escritas para serem enviadas pela rede? O objeto "João" pode ser formatado de 3 formas clássicas.

Formato JSON

{
  "nome": "João",
  "idade": 25,
  "cidade": "São Luís"
}

Linguagem moderna e compacta baseada em pares de **Chave / Valor**. Hoje é a rainha da internet.

Formato XML

<pessoa>
  <nome>João</nome>
  <idade>25</idade>
  <cidade>São Luís</cidade>
</pessoa>

Usa etiquetas (tags) com sinal de maior/menor para marcar onde começa e termina cada dado. Muito estruturado.

Formato CSV

nome,idade,cidade
João,25,São Luís

Dados dispostos em linhas simples separados apenas por **vírgulas**. É o formato nativo de planilhas Excel.

CAPÍTULO 4

Qual formato escolher?

Cada formato possui forças e fraquezas. Não existe o "melhor", existe o mais adequado para o cenário.

JSON

Moderno & Leve

  • Muito fácil de ler por humanos
  • Arquivos pequenos (economiza internet)
  • Compatibilidade nativa com JavaScript
  • Não suporta comentários internos

XML

Estruturado & Verboso

  • Extremamente robusto e rígido
  • Suporta validações complexas nativas
  • Muito verboso (repete tags toda hora)
  • Arquivos grandes e lentos para processar

CSV

Simples & Tabular

  • O menor tamanho de arquivo de todos
  • Perfeito para importar no Excel/planilhas
  • Péssimo para dados aninhados (hierarquias)
  • Não diferencia tipos de dados (tudo é texto)

CAPÍTULO 6 - PRÁTICA DE BUG-HUNTING

Regras de Sintaxe Importam!

Os computadores são muito rápidos, mas são "bobos" no quesito de sintaxe. Se você esquecer uma única vírgula ou aspas, o computador não consegue ler o objeto.

Veja o JSON no monitor de código à direita. Ele possui um erro de sintaxe. Tente validar e veja como o sistema de APIs da empresa falha por causa de um único caractere!

Painel de Validação

Clique para validar o formato enviado.

cadastro_erro.json JSON INCORRETO 
1: {
2:   "nome": "João"   <-- Falta uma vírgula aqui!
3:   "idade": 25,
4:   "cidade": "São Luís"
5: }
Pronto para validar o JSON do cadastro.

MÓDULO CONCLUÍDO

Parabéns! Você dominou JSON, XML e CSV!

Agora você já entende que um **Objeto** é uma caixa digital de atributos e que os formatos JSON, XML e CSV são apenas formas de escrever essa caixa em texto para transferi-la pela rede.

Objetos Virtuais

Atributos do mundo real empacotados em chaves e valores digitais.

Sintaxe Rígida

Uma simples vírgula ou aspas faltando quebra a comunicação (HTTP 500).

Flexibilidade

O mesmo objeto pode ser escrito como JSON, XML ou CSV.
